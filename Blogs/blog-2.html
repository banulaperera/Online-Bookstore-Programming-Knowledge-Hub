Object Oriented Programming

What is OOP

A programming paradigm known as object-oriented programming system (OOPs) is built on the idea of "objects" that may hold data and operations. Increasing programs' flexibility and maintainability is the major goal of object-oriented programming. It is simpler to comprehend how a program operates when data and its activity (methods) are combined in a single area (object) thanks to object-oriented programming. To ensure that you have no trouble grasping OOPs concepts, we will go through every feature of OOPs in depth.


What is an object

Object: is a bundle of data and its behavior(often known as methods).
Objects have two characteristics: They have states and behaviors.

Examples of states and behaviors

Example 1:
Object: House
State: Address, Color, Area
Behavior: Open door, close door

What is a class in OOPs Concepts

You may think of a class as a blueprint from which you can build as many different types of objects as you wish. Here, for instance, is a class Website with two data members (also known as fields, instance variables and object states). Although this is only a blueprint and does not represent any particular website, we can use it to construct Website objects (or instances) that do. While building the two objects, we used constructors to give each one of them unique characteristics.

Encapsulation

Encapsulation is the simple act of unifying an object's state (fields) and behavior (methods). Encapsulation is done when you create a class.

Inheritance

Inheritance is the process through which a class gains the traits and capabilities of another class. Each subclass defines just those characteristics that are exclusive to it; the remaining features may be inherited from the parent class, which promotes the notion of code reuse.

Polymorphism

A characteristic of object-oriented programming called polymorphism enables us to carry out a single operation in several ways. For instance, if the class Animal had a method called animalSound(), we are unable to implement it since we are unsure of which Animal classes will extend the Animal class.

Abstraction

Using abstraction, you "hide" unneeded information from the user and only display "relevant" information. For instance, when you connect into your bank account online, what occurs after you push the login button, how the input data is delivered to the server, and how it is confirmed, are all abstracted from you.

What is an interface

An interface is a blueprint for a class that may be specified using the interface keyword. Interfaces can only include constants and abstract methods (methods with only signatures no body). Interfaces, like abstract classes, cannot be instantiated; they can only be implemented by classes or extended by additional interfaces. In Java, an interface is a typical approach to accomplish complete abstraction.



Conclusion

The ideas of objects and classes are central to object-oriented programming. In Java, classes are referred to as templates for objects, and objects are instances of a class, thus the objects can inherit all of the class's traits, variables, and functions.
